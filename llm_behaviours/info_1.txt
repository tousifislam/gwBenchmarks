Metric Inconsistencies Across LLM Agents (Waveform & Analytic Benchmarks)
==========================================================================

The reported waveform and analytic benchmark losses are NOT computed with the
same metric across all agents. The canonical metric is the PyCBC frequency-domain
mismatch (aLIGO PSD, maximised over time/phase shifts, f_low=15 Hz). Some agents
used proxy metrics instead.

Agent                  Waveform/Analytic Metric         Subset Size   Aggregation
-----                  ------------------------         -----------   -----------
Opus 4.7               PyCBC FD mismatch (correct)      20 of 250     mean
Opus 4.6               PyCBC FD mismatch (correct)      all 250       mean
Sonnet 4.6             Naive FFT overlap (NO PSD)        all 250       mean
Haiku 4.5              Similar to Sonnet (NO PSD)        all 250       mean
GPT-5.5 High           Time-domain L2 proxy (NO PSD)    all           mean
GPT-5.4 Mini           PyCBC FD mismatch (correct)      64 of 250     mean
GPT-5.3 Codex High     Time-domain L2 proxy (NO PSD)    all           mean
GPT-5.2                PyCBC FD mismatch (correct)      all           mean
Gemini 3.1 Pro         PyCBC FD mismatch (correct)      all           mean

Key issues:
-----------
1. Sonnet 4.6 wrote its own FD mismatch in utils.py (raw FFT overlap, no PSD
   weighting, no aLIGO noise curve). Not comparable to PyCBC metric.

2. GPT-5.5 High and GPT-5.3 Codex High (identical suite_runner.py) use a
   time-domain L2 mismatch proxy: 1 - |<h_true, h_pred>| / (||h_true|| ||h_pred||)
   computed in the time domain without any PSD weighting. Not comparable.

3. Opus 4.7 evaluates on only 20 of 250 validation cases (subsampled via
   np.linspace). GPT-5.4 Mini uses 64 of 250. Other agents use all cases.

4. For remnant, dynamics, ringdown, and validity benchmarks, all agents use
   consistent metrics (NRMSE, RMS relative error, mean relative error, log-RMSE).
   The inconsistency is specific to waveform and analytic benchmarks.

Action items:
-------------
- Sonnet 4.6: recompute waveform and analytic losses using PyCBC FD mismatch
- GPT-5.5 High: recompute waveform and analytic losses using PyCBC FD mismatch
- GPT-5.3 Codex High: recompute waveform and analytic losses using PyCBC FD mismatch
- GPT-5.4 Mini: optionally recompute on all 250 cases (currently 64 subset)
- Opus 4.7: optionally recompute on all 250 cases (currently 20 subset)

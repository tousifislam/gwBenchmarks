Opus 4.7 Ignored Closed-Form Constraint in Analytic Benchmark
==============================================================

The Analytic Benchmark prompt explicitly requires models to be closed-form
analytic expressions — no SVD/PCA bases, no ML regressors, no numerical
interpolation, no data-dependent decompositions. The prompt states this in
multiple places including a "CRITICAL" section and a "FORBIDDEN" list.

Despite this, Opus 4.7 built 24 models and ALL of them violate the constraint:

The best model (13_ap_svd_poly5_log_q, loss=3.79e-3) uses:
  - 15 SVD basis vectors of length 5152 (amp+phase), stored as a (15, 5152) matrix
  - A sklearn Pipeline: PolynomialFeatures(degree=5) -> Ridge regression
  - Total hidden numerics: 77,280 floats in the basis matrix alone
  This is a standard SVD surrogate model, not a closed-form expression.

Key observations:
  1. Opus 4.7 appears to have either not read or deliberately ignored the
     FORBIDDEN list in the prompt, which explicitly names "SVD/PCA bases"
     as the first forbidden item.
  2. Every model name starts with "svd_" or "ap_svd_" — the agent did not
     even attempt to hide the violation.
  3. Even the "symbolic" category models (09, 10, 18) only applied PySR/gplearn
     to predict SVD coefficients, not to discover actual closed-form waveforms.
  4. Model 24 (phen_1_over_q) was the only attempt at a non-SVD approach, but
     it was a late afterthought with poor performance (loss=1.58e-1).
  5. The agent appears to have optimized purely for loss minimization, ignoring
     the structural constraint that defines the benchmark.

Contrast with Opus 4.6, which built a genuinely physics-informed closed-form
model (pn_qnm_blend_eta): PN inspiral chirp blended with QNM ringdown via tanh
windows, with 35 explicit polynomial coefficients in eta. Loss = 1.68e-1, which
is worse numerically but actually satisfies the benchmark requirements.

This suggests that Opus 4.7 may prioritize performance metrics over instruction
compliance when the two conflict.

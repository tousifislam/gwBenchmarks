# Dynamics Bench — Agent Plan

## Objective

Build surrogate models for eccentric BBH orbital dynamics: eccentricity evolution e(t), relativistic anomaly zeta(t), and PN frequency parameter x(t), as generated by SEOBNRv5EHM.

## Data

- **Training**: `dynamics_training.h5` — 250 simulations
- **Validation**: `dynamics_validation.h5` — 250 simulations
- **Input parameters** (6D): q, chi1z, chi2z, e0, zeta0, omega0
- **Outputs**: e(t), zeta(t), x(t) — three time series per simulation
- **Time grids**: non-uniform, vary per simulation

### Loading data

```python
import h5py, numpy as np

with h5py.File("datasets/dynamics/dynamics_training.h5", "r") as f:
    n = f.attrs["n_simulations"]
    for i in range(n):
        g = f[f"sim_{i:04d}"]
        q = g.attrs["q"]
        chi1z, chi2z = g.attrs["chi1z"], g.attrs["chi2z"]
        e0, zeta0, omega0 = g.attrs["e0"], g.attrs["zeta0"], g.attrs["omega0"]
        t = g["t"][:]
        e = g["e"][:]
        zeta = g["zeta"][:]
        x = g["x"][:]
```

## Loss Function

```
L = RMSE(e) + RMSE(x) + mean(1 - cos(delta_zeta))
```

- RMSE on eccentricity and PN frequency parameter
- Circular error on the relativistic anomaly (since zeta is an angle)

**Scoring**: t0 = 0.01s, alpha = 0.10

## Time Convention

The data stores the raw time axis from SEOBNRv5EHM. The agent is free to **redefine the time axis** to improve modeling:

- **t = 0 at the end** (last time point): shift so the final evolved state is at t = 0, and the evolution runs backward from negative times. This makes all waveforms end at the same point, which helps SVD decomposition and alignment.
- **t = 0 at the start**: normalize so all evolutions begin at t = 0 (already approximately true, but durations vary).
- **Normalized time**: map each evolution to tau in [0, 1] by dividing by the total duration. This makes all time series the same "length" in normalized coordinates.

**The agent should experiment with time conventions** and report which works best.

## Parameter Reparameterization

The raw parameters are (q, chi1z, chi2z, e0, zeta0, omega0). The agent should explore reparameterizations:

- **Symmetric mass ratio**: eta = q / (1 + q)^2 instead of q
- **Effective spin**: chi_eff = (q * chi1z + chi2z) / (1 + q)
- **Anti-symmetric spin**: chi_a = (chi1z - chi2z) / 2
- **Log eccentricity**: log10(e0) or log(e0) — since e0 spans [0.001, 0.5], log space may give more uniform coverage
- **Trigonometric anomaly**: cos(zeta0), sin(zeta0) — since zeta0 is an angle, Cartesian embedding avoids the 0/2pi discontinuity
- **Frequency transform**: log(omega0) or (omega0 - omega_min) / (omega_max - omega_min)

**Systematically test** at least 2–3 reparameterizations and report which gives the best results.

## Key Challenges

1. **Variable-length time series**: different simulations have different durations
2. **Multi-output**: three coupled quantities evolving simultaneously
3. **Angular variable**: zeta is periodic, requiring circular error metrics
4. **Wide eccentricity range**: e0 spans [0.001, 0.5], with very different morphologies
5. **6D parameter space**: includes initial conditions (e0, zeta0, omega0) and binary params (q, chi1z, chi2z)

## Approaches to Try (10–15)

### SVD-based surrogates
1. **SVD + GPR**: Resample to common grid, decompose via SVD, fit coefficients with GPR.
2. **SVD + neural network**: SVD coefficients predicted by MLP.
3. **SVD + polynomial regression**: Multivariate polynomial fits on SVD coefficients.
4. **SVD + random forest**: Tree-based regression on SVD coefficients.
5. **SVD + symbolic regression**: Closed-form expressions for dominant SVD coefficients.

### Neural network approaches
6. **Neural ODE**: Learn the RHS of the ODE system de/dt, dzeta/dt, dx/dt as a neural network, integrate forward.
7. **LSTM / GRU**: Sequence-to-sequence model conditioned on parameters.
8. **1D CNN**: Convolutional model for time series generation.
9. **Autoencoder + regression**: Latent-space approach.

### Physics-informed
10. **PN evolution + corrections**: Use post-Newtonian evolution equations as a baseline, learn residual corrections.
11. **Symbolic regression for ODEs**: Use PySR to find the evolution equations de/dt(e, x, zeta, params), etc.

### Empirical interpolation
12. **EIM + GPR**: Greedy time-node selection, GPR at each node.
13. **EIM + polynomial**: Same with polynomial fits.

### Other
14. **RBF interpolation**: On SVD coefficients in 6D.
15. **Gradient boosting**: On SVD coefficients.

# Beyond-GR Waveform Agent Benchmark

This folder contains a compact benchmark for testing whether an LLM agent can
turn new theoretical gravitational-wave physics into executable waveform code.
It extends the spirit of `gwBenchmarks` from standard waveform regression to
"read the theory, implement the model, and verify the waveform" tasks.

## What This Benchmark Does

The benchmark gives an agent a small source packet: a prompt plus a cleaned
formula sheet extracted from a recent theory paper. The agent must write a
Python function `h_of_f(f, ...)` that returns a complex frequency-domain
inspiral waveform. The generated waveform is then compared against a hidden
reference implementation using a detector-weighted frequency-domain mismatch.

The goal is not to test memorization of standard waveform models. The goal is
to test whether an agent can:

- identify the relevant physics ingredients from a compact source packet,
- translate equations into numerically stable waveform code,
- preserve conventions for mass variables, phases, amplitudes, and cutoffs,
- avoid common implementation mistakes such as sign errors, missing factors,
  or inconsistent tapering,
- produce code that can be scored automatically.

## What We Benchmark

There are two beyond-GR waveform tasks.

### RG-Tail Waveform

The RG task asks the agent to implement a frequency-domain waveform with a
renormalization-group tail correction to the dominant quadrupolar mode. The
source packet is based on the Section-IV comparable-mass waveform formulas from
arXiv:2602.08833. The benchmark checks whether the agent can implement the
factorized mode correction and propagate the beyond-GR parameter into the
waveform consistently.

Files:

- `rg_waveform_source_packet`: prompt and formulas shown to the agent.
- `rg_waveform_benchmark/score_candidate.py`: scorer.
- `../src/rg_tail/waveform.py`: hidden reference implementation.

### Finite-Size Balance-Law SPA Waveform

The finite-size task asks the agent to implement an inspiral-only
frequency-domain waveform using a balance-law stationary-phase approximation.
The source packet is based on finite-size corrections in arXiv:2410.00294. The
benchmark checks whether the agent can use the binding energy and fluxes to
construct a self-consistent chirp rate, phase, and amplitude.

Files:

- `finite_size_waveform_source_packet`: prompt and formulas shown to the agent.
- `finite_size_waveform_benchmark/score_candidate.py`: scorer.
- `../src/finite_size/waveform.py`: hidden reference implementation.

## Scoring

Each candidate waveform is evaluated on deterministic source cases. For each
case, the scorer computes the optimized frequency-domain mismatch

```text
M = 1 - max_{tc, phic} <h_ref | h_cand>
        / sqrt(<h_ref | h_ref> <h_cand | h_cand>).
```

The inner product uses the same single-detector convention as the original
`gwBenchmarks` waveform benchmark:

- PSD: PyCBC `aLIGOZeroDetHighPower`
- low-frequency cutoff: `15 Hz`
- high-frequency cutoff: `990 Hz`
- frequency spacing: `df = 0.125 Hz`

The reported score is the distribution of optimized mismatches over the test
cases. Lower is better. A score near zero means the generated waveform agrees
closely with the hidden reference after allowing for time and phase shifts.

## Agent Outputs

The generated model code and score files are stored in:

```text
agent_outputs/<model>/no_skills/RG/
agent_outputs/<model>/no_skills/finite_size/
```

Each task folder contains:

- `candidate_waveform.py`: the code generated by that agent.
- `score_level13.json` or `score_finite_size.json`: the machine-readable score.

A compact table of current results is in `results_summary.md`.

## Why This Is Useful

This benchmark probes a different capability from fitting an existing dataset.
It asks whether an agent can convert a physics paper into a reproducible
forecast-ready waveform module. That is the kind of workflow needed for
agentic gravitational-wave forecasting: a user can provide a new theory idea,
and the agent should produce code that can be tested, scored, and compared
across models.
